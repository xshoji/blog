はじめに

最近、 AmpCode に隠しモードが追加された（らしい）

Large Mode - Amp
https://ampcode.com/news/large-mode

その名も large モード。このモードは、主に大きいコンテキストを対象にしたコーディングで使うことを想定してるみたい。また、まだ検証段階らしく、積極的に使って欲しい意図はない、との書かれてある。

Use large mode sparingly. Using it never is totally fine, and we made this news post undiscoverable since we don’t want most people stumbling across it and using it.
largeモードは必要最小限の使用にとどめてください。このモードを「全く使用しない」のも全く問題ありません。むしろ、ほとんどのユーザーが偶然この機能を見つけて使用することを避けるため、このニュース記事は意図的に見つけにくくしています。

一方で、このlargeモードは 2026-03-19(木) 現在、 Claude Sonnet 4.6 が使われる。 Claude Sonnet 4.6 は、 Claude Opus(オーパス) 4.6 と比べて、同等のコーディング性能を持ちながらコストを抑えることができるらしく、高い性能でありながらコストパフォーマンスが良いことが予想される。

モデル入力（Input）出力（Output）
Sonnet 4.6$3 / 100万トークン$15 / 100万トークン
Opus 4.6$5 / 100万トークン$25 / 100万トークン

ちなみに、冒頭のampのお知らせを見ると、 large モードをコスパを目当てに使うなと書かれてる。…が、まぁ、使えるは使えるので、どの程度の使用感でコスパなのか、以前の調査

AmpCodeの広告付き無料プランの1日のクレジットでできること:smartモードとrushモードの比較
https://blog.xshoji.com/posts/20260214_ampcode-free-1day-challenge/

と同じことをやらせてみた。

実装させてみた所感

前回の流れと同じように、要件をまとめたmdファイルを読み込ませつつ実装計画をたてさせて、フェーズごとに実装を進めてもらった。早速結果はこちら

まず、不具合が圧倒的に少なかった。また、何も言わずとも最初からいい感じのデザインを出してきて、特に修正依頼を出さずに済んだ。ここで注意が必要なのは、前回調査したとき smart モードはまだ Opus 4.5 だったので、この辺は単純にモデルの性能が上がって改善されたんだと思う。現在 smart モードは Opus 4.6 になっており、さらに高い品質でサイトを作れそう。そっちはやらせてないけど（じゃあ比較になってないやん、というのはそれはそう）

で、普段 smartモードの Opus 4.6 を使っているが、それと比べてコストが少なく抑えられてるのははっきり感じた。rushモード（現在は Claude Haiku 4.5）は明らかに頭が悪い間違いや同じミスを繰り返すが、largeモードはそういうのがない上コストが低めで、ちょうどsmartとrushの間（smartより）という感じでした。個人的には普段使いを large モードにしたくなっちゃったけど、公式の案内では実験的モードなので積極的には使わないで欲しそう🤔

2026-03-19現在、 large コンテキストじゃなくなってる

Set "amp.internal.visibleModes": ["large"] in Amp settings. Then the large mode will be Sonnet 1M context. Remove the useSonnet flag.

— Quinn Slack (@sqs) January 6, 2026

で amp の CEO が言ってるが、以前は 1M contextへの対応をうたっていた。実際、数日前に使った時は Maxのコンテキストサイズが 986k と表示されており、かなりでかいコンテキストを扱える状態になっていた。

ただ、今日試したところ

89% of 168k

のように、1M コンテキストじゃなくなってた（smartと同じコンテキストサイズ）。もしかしたらバグかもしれないが、これだと全然 large じゃないやんと思った。

2026-03-24(火) 追記: その後、再度試したところ、コンテキストサイズが 986k と表示されるようになっていた。なのでバグだったっぽい。
